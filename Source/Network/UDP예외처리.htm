* UDP 예외 처리

 예시 )            CN <----X---- CS               CN이 못 받는 경우

 1. MS는 CS와 연결상태 확인. 연결이 끊긴 경우 CS의 상태 점검을 시작한다. ( 프로토콜 필요)

     TCP 및 UDP 확인 필요

 2. 1번의 연결은 되어 있으나 해당 CN이 CS와 접속이 안되는 경우

    - MS에게 접속불가를 알려준다.

    - 접속불가 수가 과반수 이하인경우 접속불가 CN의 접속을 끊어준다.

    - 모두 접속불가인경우는 강제로 CS를 끊어주고 새로운 CS를 설정해준다.

 3. 1번의 연결이 안되면 CS의 Session 종료 및 새로운 CS를 설정한다.

 

 예시 )             CN ----X---->CS               CS가 CN으로부터 못 받는경우

 1. CN에 대한 접속여부를 MS에게 알려준다.

 2. MS는 해당 CN에 대해 상태 점검.

 3. 연결이 되어 있다면 CN에게 접속불가를 통보

 4. 연결이 안되면 Session 종료
